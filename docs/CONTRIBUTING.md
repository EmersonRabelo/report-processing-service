# Contribuindo

Obrigado por considerar contribuir! Seguem diretrizes básicas:

1. Abra uma issue descrevendo a mudança desejada antes de grandes alterações.
2. Crie branches com nomes descritivos: `feat/<tema>`, `fix/<bug>`, `chore/<tarefa>`.
3. Faça commits pequenos e com mensagens claras.
4. Adicione testes sempre que possível.
5. Envie um Pull Request descrevendo o que foi alterado e por quê.

Estilo de código:

- Siga `gofmt`/`gofumpt` para formatação.
- Siga convenções do Go para nomes e documentação.
